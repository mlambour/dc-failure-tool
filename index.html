🧾 Executive Summary

📘 Assumptions & Methodology

Show calculation assumptions
  • UPS sizing is based on IT load only; mechanical, pumps, and miscellaneous loads are fixed non-IT loads per string.
  • Mechanical / cooling / pump / miscellaneous loads do not redistribute during failure scenarios.
  • UPS input power is calculated from UPS output plus configured charging and UPS losses.
  • Generator sizing is based on worst-case surviving string demand including UPS input and fixed non-IT loads, converted by entered power factor.
  • Transformer sizing is selected from the entered transformer rating list using the worst-case apparent power plus entered infrastructure loss allowance.
  • Dual mode assumes 50/50 split with survivor taking 100% on failure.
  • Triple mode assumes two active feeds at 50/50 and one catcher that absorbs the lost 50% share on active-feed failure.
  • Quad mode assumes two independent paired paths (A/B and C/D), each redistributing only within its pair.
  • Quint mode assumes four active feeds at 25% each plus one catcher that absorbs only the failed active 25% share.
  • Validation warnings are advisory checks only: load imbalance flags uneven normal IT distribution, governing failure uplift flags when the worst failure case materially exceeds normal loading, PF / loss warnings flag unusual design inputs, transformer list validation checks that a usable rating schedule exists, and duplicate feed warnings flag repeated string assignments within a single row.
